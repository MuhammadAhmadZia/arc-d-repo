# Results

Every CSV in this directory is regenerated by the scripts in `scripts/`. None of
these files are the input to any code; they exist so that anyone reading the
paper can look up an exact number without rerunning the experiments.

## `final_results.csv`

One row per (dataset, seed, system, condition). Populated by `eval_all.py`.

| Column | Meaning |
|---|---|
| dataset | `gsd` or `freiburg` |
| seed | training seed (0, 1, or 2) |
| system | `cheap-B0`, `distilled-B0`, `heavy-Tiny`, `ARC`, or `ARC-D` |
| condition | `clean`, `jpeg_s2`, `blur_s2`, or `occ_s1` |
| acc | test accuracy on this condition, in [0, 1] |
| gmacs | average GMACs per image for this system on this condition |
| routed | fraction of test images routed to stage 2 (ARC and ARC-D only) |
| tau | routing threshold used (ARC and ARC-D only), selected on val |

This is the source of truth for Tables 2 and 3 in the paper.

## `dual_confidence_results.csv`

The base-paper mechanism as an evaluated baseline. One row per (dataset, seed,
condition). Populated by `dual_confidence.py`. Same schema as above minus
`routed` and `tau`.

## `hw_results.csv`

One row per (target runtime, metric). Populated by `hw_measure.py`. This is the
source of Table 5.

| Column | Meaning |
|---|---|
| timestamp | when the row was collected (informational) |
| env | detected platform: `colab`, `kaggle`, or `other` |
| device | GPU name (e.g. `Tesla T4`) or CPU model with core/thread count |
| metric | `GMACs`, `gpu_latency_ms`, `gpu_energy_kgco2e`, or `cpu_latency_ms` |
| cheap | measured stage-1 (EfficientNet-B0) value |
| heavy | measured stage-2 (ConvNeXt-Tiny) value |
| ratio | `cheap / heavy`; the cascade saves iff `ratio < 1 - f` |
| save_fXX | percent saving at routed fraction `XX` (0.30, 0.40, 0.43, 0.60) |

## A note on comparing energy across devices

Do not compare absolute energy readings across different GPU or CPU rows. Only
the within-device `ratio` is used for any claim in the paper. Trackers sample
power differently across environments, so absolute values are not directly
comparable, but the ratio of two forward passes measured on the same device
under the same protocol is trustworthy.
